# Omnisense AI: Features Implemented in Unity (v1.0)

This document serves as a technical record of the features, architectural decisions, and "SOTA" (State of the Art) UX enhancements implemented during the Unity port.

---

## 1. Core Architecture: The Autonomous ReAct Loop
**What:** A recursive **Reason-Act-Observe** pipeline that allows the agent to function as a truly autonomous developer.
**Why:** Standard "one-shot" chat interfaces require constant user hand-holding. ReAct allows the agent to think, execute a tool, see the result, and iterate until the task is complete.
**How:**
- **Recursive Orchestration**: The `AIOrchestrator` uses a callback-based loop. When an LLM response contains a tool call, the system executes it and feeds the result back into the prompt automatically.
- **Thought Parsing**: Mandatory `<thought>` blocks are extracted and styled uniquely to separate the agent's internal reasoning from its final output.
- **Tool Hiding**: Raw JSON-RPC (`mcp_json`) blocks are stripped from the chat UI after execution to keep the history clean and readable.

---

## 2. Unity-Specific Tooling (MCP Tools)
The agent's "hands" have been specifically tuned for the Unity Editor environment.

### A. Scene Manipulation & Reflection
- **`scene/instantiate_node`**: Creates GameObjects.
    - *How*: Supports primitive types (Cube, Sphere) and empty GameObjects. 
    - *Fix*: Optimized for synchronous execution to avoid **Main Thread Deadlocks** during the ReAct loop.
- **`scene/modify_node` (Component Injection)**: Allows the agent to build functional objects.
    - *How*: Added an `add_component` property. It uses **C# Reflection** to scan all loaded assemblies (including `Assembly-CSharp`) to find and attach components by name (e.g., `Rigidbody2D`, `BoxCollider2D`, or custom user scripts).
- **`scene/inspect_node` (Scene Reading)**:
    - *Why*: The agent was "blind" to existing objects.
    - *How*: Returns the exact position and a full list of all attached components on any GameObject, allowing the agent to diagnose missing colliders or incorrect script assignments.

### B. Filesystem & Scripting
- **`project/write_file`**: Creates C# scripts.
    - *Integration*: Triggers `AssetDatabase.Refresh()` automatically.
    - *UI Logic*: Detects when Unity is compiling and informs the user via the chat status.

## 3. Premium UX & Interface (Unity UI Toolkit)

### A. Resizable Workspace
- **`TwoPaneSplitView`**: Implemented a modern, flexible layout where users can vertically resize the ratio between Chat History and the Plan/Context area.

### B. Ephemeral Context Engine
- **Per-Turn Context**: Chips are automatically cleared after a message is sent (matching Cursor's logic) to prevent "context pollution" in long sessions.
- **Drag & Drop**: Direct integration with Unity's `DragAndDrop` API.
- **Smart Paste (Ctrl+V)**: The input field intercepts the clipboard. If a valid file path is detected, it converts it into a Context Chip instead of raw text.
- **Attachment Button**: Added a Paperclip (📎) button using `EditorUtility.OpenFilePanel` for manual file selection.

### C. Interactive Chat History
- **Inspector Pinging**: Context chips are "baked" into chat messages. Clicking a resource name in a chat bubble (even in history) instantly pings and selects that object in the Unity Inspector.
- **Word Wrap**: All AI responses, code blocks, and tool observations feature full horizontal word-wrapping to ensure readability in narrow docked windows.
- **Animated Spinner**: An ASCII carousel (⠋ ⠙ ⠹) appears while the agent is thinking, providing visual feedback that the ReAct loop is active.

---

## 4. Stability & Persistence
- **Model Persistence**: Selected AI models are saved via `EditorPrefs`, ensuring your choice (e.g., Gemini vs GPT) stays selected after restarting Unity.
- **Path Resolution Fix**: Implemented a "Path Stripper" that handles the leading slashes (e.g., `/Player`) generated by our hierarchy-path logic, ensuring `GameObject.Find` works reliably for root objects.
- **Safe Reflection**: Assembly scanning for components is wrapped in try-catch blocks to prevent internal Unity assemblies from crashing the tool-discovery process.

---

## 5. SOTA Agent Capabilities (Phase 4)
- **Console Log Interception**: The plugin now hooks into `Application.logMessageReceived` to maintain a 30-log ring buffer of Errors and Warnings. The agent can use the `editor/read_console` tool to autonomously diagnose compiler errors or runtime crashes.
- **Asset Metadata Inspection**: Added `project/inspect_asset`. This allows the agent to bypass binary data limitations and read native Unity metadata (Components, Material properties, Texture dimensions) using `SerializedObject` reflection.
- **Safe File Context**: Implemented `project/read_file` to ensure the agent can safely ingest existing scripts without the risk of accidental overwrites during inspection.
- **Per-Turn Inline Undo**: The Undo system is now context-aware. AI responses include an inline **"↺ Undo Turn Actions"** button that reverses only the specific modifications (file edits or scene changes) made during that individual chat turn.

---

## 6. Dynamic Scene & Project Wiring (Phase 5)
- **Dynamic Property Modification**: Added `scene/set_component_property`. The agent can now modify any serialized field on any component, including primitives, booleans, and Vector2/3 types.
- **Smart Object Linking**: The agent can now "wire" objects together. If a script field (e.g., `Target`) expects a Transform or GameObject, the agent can pass a name (e.g., `"Player2D"`), and the tool will automatically resolve the reference and link the objects.
- **Smart Asset Search**: All scene tools now incorporate `AssetDatabase.FindAssets` logic. The agent no longer needs exact paths; it can find and modify Prefabs, Materials, and Textures anywhere in the project folder just by mentioning their name.
- **Prefab Authoring**: The agent can now modify Prefabs directly in the project folder without needing to instantiate them in the active scene.

---

## 7. Project DNA: Persistent Meta-Memory (Phase 6)
- **Long-Term Memory**: Implemented a `.omnisense_dna.md` system. The agent now carries architectural knowledge across editor restarts.
- **DNA Bootstrapping**: At the start of every session, the orchestrator injects the contents of the DNA file into the system context, ensuring zero-shot conformity to project rules.
- **Autonomous Learning**: Added `project/update_dna`. The agent is mandated to silently update this file whenever it learns new project-specific conventions, folder structures, or architectural patterns.
- **User-Editable Logic**: Since DNA is stored as a simple Markdown file in the root, users can manually "train" the agent by adding bullet points to the file.

---

## 8. Trust Framework: The Diff Illusion (Phase 7)
- **Asynchronous Execution Pausing**: Refactored the `AIOrchestrator` ReAct loop to pause execution whenever the agent attempts a "destructive" action (e.g., `write_file`, `modify_node`, `instantiate_node`).
- **Visual Staging UI**: The UI now intercepts these actions and renders a rich-text "Diff Summary" bubble (e.g., `<color=#00FF00>+ Add Component: Rigidbody</color>`).
- **Interactive Gatekeeping**: Users are presented with `[✓ Accept]` and `[✗ Reject]` buttons. This builds psychological safety, allowing senior engineers to trust the agent with complex, multi-file tasks.
- **Rejection Feedback Loop**: Clicking `[Reject]` feeds an observation back to the agent ("User rejected this change"), forcing the AI to autonomously pivot, ask for clarification, or try a different approach.

---

## 9. Context Optimization (Phase 8)
- **Ephemeral Tool Observations**: To prevent "Lost in the Middle" context bloat during long sessions, the orchestrator now actively prunes the context window. Once a ReAct loop completes, large `[Observation]` blocks (like 1000-line script reads) are truncated.
- **System Prompt Re-Injection**: The orchestrator now injects a hidden format reminder at the very end of the payload array immediately before querying the API. This guarantees that strict JSON formatting instructions remain in the model's primary attention span, completely eliminating formatting hallucinations caused by prompt saturation.
- **Robust Tool Extraction**: Upgraded the regex logic to use a three-tier fallback system, allowing the orchestrator to extract tool calls even if the LLM omits markdown closing ticks or outputs raw JSON.
